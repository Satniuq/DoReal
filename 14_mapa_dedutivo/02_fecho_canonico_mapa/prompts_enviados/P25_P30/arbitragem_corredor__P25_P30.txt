CRITÉRIO DE FECHO DO CORREDOR

Considera o corredor fechado apenas se:
- todos os passos indispensáveis da sequência mínima estiverem fechados;
- as pontes de entrada e saída estiverem coerentes ao longo da sequência;
- as objeções letais relevantes estiverem bloqueadas;
- os subpassos necessários estiverem aprovados e integrados;
- não restar salto dedutivo impeditivo entre passos consecutivos.

Considera o corredor parcial quando:
- há progresso local forte, mas permanece pelo menos uma lacuna impeditiva;
- algum passo está quase fechado mas ainda sem mediação suficiente;
- há formulações promissoras, mas ainda não fixáveis.

Considera o corredor para reabrir quando:
- a sequência mínima ainda falha em pontos decisivos;
- há passos que precisam de revisão local antes de qualquer consolidação;
- a arbitragem não pode fixar as formulações com segurança.

REGRAS

- Reabre apenas os passos estritamente necessários.
- Aprova apenas os subpassos realmente indispensáveis.
- Fixa apenas formulações que já podem ficar estáveis.
- Não fixes formulações provisórias como finais.
- Se o corredor estiver fechado, a lista de passos_a_reabrir deve ser vazia.
- Não escrevas texto fora do JSON.

FORMATO DE SAÍDA

Responde exclusivamente com um único objeto JSON válido.
Usa exatamente estas chaves e esta estrutura:

{
  "tipo_registo": "arbitragem_corredor",
  "corredor": "P25_P30",
  "sequencia_minima_do_corredor": [
  "P25",
  "P26",
  "P27",
  "P28",
  "P29",
  "P30"
],
  "decisoes_por_passo_referenciadas": ["P.."],
  "decisoes_por_subpasso_referenciadas": ["P.._SP.."],
  "passos_fechados": ["P.."],
  "passos_quase_fechados": ["P.."],
  "passos_a_reabrir": ["P.."],
  "subpassos_aprovados": ["P.._SP.."],
  "formulacoes_fixadas": [
    {
      "passo_id": "P..",
      "formulacao_canonica_final": "string"
    }
  ],
  "sequencia_minima_ficou_fechada": true,
  "estado_final_do_corredor": "fechado|parcial|reabrir",
  "criterio_de_fecho_aplicado": "string",
  "justificacao_da_arbitragem": "string",
  "pendencias_remanescentes": ["string"],
  "fontes_utilizadas": ["dados/...json", "../.../ficheiro.json"],
  "observacoes_editoriais": ["string"]
}

REGRAS DE PREENCHIMENTO DO JSON

- "decisoes_por_passo_referenciadas" deve listar os passos realmente usados na arbitragem.
- "decisoes_por_subpasso_referenciadas" deve listar apenas os subpassos efetivamente considerados.
- "passos_fechados" inclui apenas passos com fecho suficientemente estável.
- "passos_quase_fechados" inclui apenas passos com avanço real mas ainda não totalmente fixados.
- "passos_a_reabrir" inclui apenas os passos cujo estado impede o fecho do corredor.
- "subpassos_aprovados" inclui apenas subpassos indispensáveis já validados.
- "formulacoes_fixadas" inclui apenas formulações que podem ser tomadas como finais dentro deste corredor.
- Se "sequencia_minima_ficou_fechada"=true, então:
  - "estado_final_do_corredor" tem de ser "fechado";
  - "passos_a_reabrir" tem de ser [];
  - "pendencias_remanescentes" deve ser [] ou conter apenas notas não impeditivas.
- Se "estado_final_do_corredor"="reabrir", então "passos_a_reabrir" tem de conter pelo menos um passo.
- "justificacao_da_arbitragem" deve ser concreta e local ao corredor.
- "observacoes_editoriais" deve conter apenas notas operacionais curtas.
