4. ponte de saída para o passo seguinte;
5. razão pela qual o anterior não basta;
6. razão pela qual este passo não pode ser suprimido;
7. objeção letal mais forte a bloquear;
8. bloqueio curto dessa objeção;
9. decisão sobre necessidade de subpasso;
10. lista final de objeções já bloqueadas;
11. lista final de fragmentos efetivamente selecionados;
12. nota final curta e operacional;
13. pendências reais, se ainda existirem.

REGRAS DE DECISÃO

- Mantém a formulação atual apenas se ela já fechar o passo sem salto relevante.
- Reformula quando a formulação existente é insuficiente, ambígua ou demasiado larga.
- Densifica quando o núcleo do passo está certo mas a justificação ainda é curta.
- Introduz subpasso apenas se o passo não fechar com simples reformulação e justificação.
- Reabre apenas se houver insuficiência que não possa ser resolvida localmente com segurança.
- A formulação final deve ser curta, afirmativa e dedutivamente precisa.
- A justificação deve dizer o mínimo suficiente para o passo ficar inevitável na cadeia.
- A ponte de entrada deve ligar explicitamente o passo ao anterior.
- A ponte de saída deve ligar explicitamente o passo ao seguinte.
- A objeção letal deve ser a mais destrutiva para este passo, não uma objeção lateral.
- O bloqueio curto deve responder diretamente à objeção letal sem deriva.
- Usa apenas ids de fragmentos efetivamente presentes nos materiais.
- Se não houver pendências, devolve um array vazio.
- Se precisa_de_subpasso=false, subpassos_aprovados_ids tem de ser [].
- Se o passo ficar fechado, pendencias deve ser [] ou conter apenas notas não impeditivas.
- Não escrevas introduções, comentários externos, markdown nem texto fora do JSON.

REGRAS DE PREENCHIMENTO DO JSON

- "formulacao_v2_final": uma só formulação final, sem variantes.
- "justificacao_expandida_final": texto suficientemente completo para fechar o passo localmente.
- "objecoes_bloqueadas_final": inclui a objeção letal e outras objeções diretamente neutralizadas pela formulação e justificação finais.
- "fragmentos_selecionados_finais": apenas ids realmente usados para sustentar a decisão.
- "fontes_utilizadas": lista apenas os ficheiros efetivamente utilizados.
- "nota_editorial_final": nota operacional curta sobre o estado do passo.
- "pendencias": só inclui insuficiências reais que impeçam ou enfraqueçam o fecho.
- Se "decisao_editorial"="introduzir_subpasso", então "precisa_de_subpasso" tem de ser true.
- Se "decisao_editorial"="reabrir", então "reabrir_em_arbitragem_de_corredor" tem de ser true.
- Se "estado_final_do_passo"="fechado", o conjunto do JSON deve mostrar fecho local claro e não deixar lacunas explícitas.
